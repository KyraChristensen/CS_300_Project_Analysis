# CS_300_Project_Analysis
Portions of Project One and Project Two

What was the problem you were solving in the projects for this course?
>The problem isn't just "sorting a list"; it’s managing computational complexity. As data grows, naive solutions (like nested loops) hit a wall where they become too slow or consume too much memory. I was solving the challenge of how to store, retrieve, and manipulate data efficiently as the input size (n) increases.

How did you approach the problem? Consider why data structures are important to understand.
>Approaching these problems requires moving beyond simple variables. Understanding data structures is vital because they are the "blueprints" for how information is organized in memory.

   >Logic over Syntax: I began by analyzing the requirements (e.g., Do I need fast lookups? Do I need to preserve order?) before writing a single line of code.

  >The Right Tool: Using a Hash Table for O(1) lookups versus a Linked List for O(1) insertions changes the entire performance profile of the app.

How did you overcome any roadblocks you encountered while going through the activities or project?
> Using print statements, debuggers, and drawing out pointer logic on paper helped overcome roadblocks. As well as breaking the problem into smaller manageable pieces. 

How has your work on this project expanded your approach to designing software and developing programs?
> I've learned that you can often make an algorithm faster by using more memory, or save memory by acceptin a slower runtime. I now design software with the "worst-case scenario" in mind rather than just the "happy path". 

How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?
>Using meaningful variable names and clear comments so the logic is self-explanatory, and using classes and data structures to make the code adaptable.
